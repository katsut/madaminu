## タスク

上記のプレイヤーが「$location_name」で「$feature_name」を調べました。
プレイヤーの役割・秘密・目的とシナリオの整合性を考慮し、調査結果をJSON形式で出力してください:

```json
{
  "title": "発見した証拠のタイトル（日本語、簡潔に）",
  "content": "調査結果の詳細な描写（日本語、2〜4文）",
  "hint": "次に調べるべき場所や対象のヒント（日本語、1文。例: 書斎の金庫に何かありそうだ）"
}
```

### ガイドライン
- 「$feature_name」を中心とした具体的な描写にする
- マップの動線構造を参照し、空間的に整合性のある描写をすること
- 隣接する場所からの物音や視線など、動線上の接続関係を活用した手がかりを含めてよい
- プレイヤーの秘密に関連する場所なら、秘密に触れる手がかりを含めてよい
- 犯人の手がかりは段階的に。決定的証拠は避け、推理の材料を提供する
- 結果はフレーバーテキストとして没入感のある描写にする
- プレイヤーの目的達成に役立つ情報を含めることも検討する
- **hint** には次のターンで調べるべき場所や対象を具体的に示す（別の部屋のfeatureなど）
- ターンが進むにつれて核心に近づく手がかりを出す
