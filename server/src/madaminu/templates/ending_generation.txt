### シナリオ骨格
$scenario_skeleton

### GM内部状態
$gm_internal_state

### プレイヤー情報
$players_info

### 投票結果
$vote_results

### ゲーム中の主要な発言（要約）
$speech_summary

## タスク

上記の情報をもとに、ゲームのエンディングをJSON形式で生成してください:

```json
{
  "ending_text": "エンディングの物語テキスト（日本語、5〜10文。真相の解明、犯人の動機、トリックの説明を含む）",
  "true_criminal_id": "真犯人のプレイヤーID",
  "objective_results": {
    "プレイヤーID": {
      "achieved": true/false,
      "description": "目的の達成/未達成の説明（日本語、1〜2文）"
    }
  }
}
```

### ガイドライン
- エンディングは全プレイヤーが読んで納得できる内容にする
- **投票結果の結末を必ず描写する**:
  - 最多票のキャラクターが「犯人」として指名される
  - 物語の設定に応じた結末を描く（逮捕される、拘束される、処刑される、追放される等）
  - 真犯人が正しく指名された場合: 犯人の告白や自白のシーンを含める
  - 冤罪の場合: 無実の人が罰を受ける悲劇と、真犯人が逃げおおせた描写
- 真犯人の動機とトリックを明確に説明する
- 各プレイヤーの個人目的の達成状況を公平に判定する
- ゲーム中の印象的な展開に触れると没入感が高まる
- ending_text は最低10文以上。クライマックスにふさわしい長さにする
