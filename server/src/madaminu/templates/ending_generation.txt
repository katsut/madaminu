## タスク

上記の情報をもとに、ゲームのエンディングをJSON形式で生成してください:

```json
{
  "ending_text": "エンディングの物語テキスト（日本語、5〜10文。真相の解明、犯人の動機、トリックの説明を含む）",
  "true_criminal_id": "真犯人のプレイヤーID",
  "objective_results": {
    "プレイヤーID": {
      "achieved": true/false,
      "description": "目的の達成/未達成の説明（日本語、1〜2文）"
    }
  }
}
```

### ガイドライン

エンディングの構成は以下の順番で描写する:

1. **監禁**: 投票で最多票を集めたキャラクターが部屋に監禁される場面
2. **真相の解明**: 監禁された人物が真犯人なのか、それとも冤罪なのかが明らかになる
   - **真犯人を当てた場合**: 犯人が観念して動機・トリックを告白する。その後の処遇を描く
   - **冤罪の場合**: 無実の人物が監禁された悲劇。真犯人はその隙に逃走、あるいは正体を現す
3. **各キャラクターのその後**: 事件後の各人の運命を簡潔に描写

- エンディングは全プレイヤーが読んで納得できる内容にする
- 真犯人の動機とトリックを明確に説明する
- 各プレイヤーの個人目的の達成状況を公平に判定する
- ゲーム中の印象的な展開に触れると没入感が高まる
- ending_text は最低15文以上。クライマックスにふさわしい長さにする
