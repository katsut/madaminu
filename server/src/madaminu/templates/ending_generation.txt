## タスク

上記の情報をもとに、ゲームのエンディングをJSON形式で生成してください:

```json
{
  "ending_text": "表のエンディング（全プレイヤー向け）",
  "criminal_epilogue": "犯人視点のエピローグ（真相解明パート）",
  "true_criminal_id": "真犯人のプレイヤーID",
  "objective_results": {
    "プレイヤーID": {
      "achieved": true/false,
      "description": "目的の達成/未達成の説明（日本語、1〜2文）"
    }
  }
}
```

### ガイドライン

エンディングは **2段構成** で描写する:

#### 第1幕: ending_text（表のエンディング）

1. **監禁**: 投票で最多票を集めたキャラクターが部屋に監禁される場面
2. **表面的な結末**: 監禁された人物の反応。他の参加者たちの安堵や不安
   - 真犯人を当てた場合でも、この時点では確定的な描写をしない
   - 冤罪の場合: 無実の人物が監禁される不穏な空気。何かが間違っているという違和感
3. **各キャラクターのその後**: 事件後の各人の運命を簡潔に描写

#### 第2幕: criminal_epilogue（犯人視点の真相）

ここで初めて真相が明かされる。犯人の一人称視点で語られるエピローグ:

1. **犯人の内心**: 投票結果を聞いた時の犯人の心境
   - 自分が指名された場合: 観念、あるいは最後の抵抗
   - 別の人が指名された場合: 安堵、罪悪感、あるいは冷笑
2. **犯行の真相**: 動機、トリック、犯行の手順を犯人の記憶として描写
3. **犯人の結末**: 逃げおおせたのか、それとも真実が暴かれたのか
   - 冤罪の場合: 後味の悪い結末。犯人は平然と去っていく。「本当の犯人は、まだこの中にいるのかもしれない」という不穏な余韻

### 目的達成の判定基準

- **犯人の目的**: 投票結果だけで判定しない。ゲーム中の行動を見る:
  - 発言で疑惑をそらせたか（他人に票を向けさせる発言をしたか）
  - 証拠を改ざんできたか
  - 自分の秘密を隠し通せたか
  - 犯人固有の目的（証拠隠滅、共犯者保護等）を達成したか
  - **投票で指名されても、上記の行動ができていれば部分的に達成とする**
- **犯人以外の目的**: それぞれの個人目的に照らし合わせ、発言内容や調査行動から判定
- **全員共通**: ゲーム中の発言内容（speech_summary）を根拠にすること。発言していない行動は評価できない

### その他

- ending_text, criminal_epilogue はそれぞれ最低10文以上
- criminal_epilogue は犯人の一人称で書く（「私は...」「俺は...」）
- ゲーム中の印象的な展開に触れると没入感が高まる
