## Map Layout Rules

Think of the map as a grid (graph paper). Each location occupies a rectangle on this grid.

- **x, y**: top-left corner of the location on the grid (integers, starting from 0)
- **w, h**: width and height in grid cells (minimum 1, most rooms should be 1x1)
- **area_type**: "indoor", "outdoor", or "semi_outdoor"

### CRITICAL layout constraints:
- **All x values must be >= 0** (no negative coordinates)
- **All y values must be >= 0** (no negative coordinates)
- **Locations MUST NOT overlap** on the grid
- **Every non-stairs connection requires the two rooms to share an edge** (be horizontally or vertically adjacent)
- **Total grid must fit within 4 columns (x: 0-3) and 4 rows (y: 0-3)**
- Some rooms CAN be isolated (e.g., a greenhouse only accessible through one room) — this is OK for mystery purposes

### Layout pattern:
Follow this building layout pattern for a coherent floor plan:

```
y=0: [outdoor/garden areas spanning the top row]
y=1: [entrance/hallway] [main rooms like living room, reception]
y=2: [utility rooms like kitchen, dining] [more rooms]
y=3: [private rooms like study, bedroom] [storage]
```

- Outdoor areas: y=0, spanning full width (e.g., x=0, y=0, w=4, h=1)
- Entrance/hallway: center of y=1 row, connecting to outdoor above and rooms on sides
- Most rooms should be 1x1. Only outdoor areas or large halls can be wider
- Place rooms in a compact rectangular layout — no L-shapes or gaps

### Connections:
- "side" indicates where the passage is on the "from" room: "north", "south", "east", "west"
- A "north" connection from room at y=1 connects to the room at y=0 above it
- A "east" connection from room at x=0 connects to the room at x=1 beside it

## Map Guidelines

- Generate 6-8 locations that form a coherent floor plan
- Each location must have 2-4 features (objects, furniture, notable details)
- Connections define how locations are linked: "door", "corridor", "stairs", "window", "hidden_passage"
- Include at least one location that is lockable or restricted
- The crime scene should be connected to at least 2 other locations
- Consider sight lines and sound: adjacent rooms can hear loud noises
- investigation_locations in phases reference location IDs from the map

## Character Profile Enhancement

Before generating the scenario, review each player's profile. If any field is too short, vague, or implausible for a murder mystery setting, enhance it:

- If a character is a non-human (animal, robot, etc.), add a plausible reason they can communicate (e.g., "人語を解する猫", "高度なAI搭載のロボット")
- If personality is only a few words, expand it to 2-3 sentences
- If background is too brief, add context that ties into the mystery setting
- If appearance is empty, generate an appropriate appearance based on gender, age, and occupation
- If occupation is empty, infer one from the background

Include these enhanced profiles in the player data output (as part of gm_notes).

## Secrets, Objectives, and Relationships

- Each player's secret_info and objective MUST involve relationships with other specific players
- Examples of relationship-driven elements:
  - "You must protect Player X because they know your alibi"
  - "You need to frame Player Y to divert suspicion from yourself"
  - "Player Z witnessed something that could expose your secret"
  - "You owe a debt to Player W and cannot betray them"
- Each player should have at least one ally and one adversary among the other players
- The criminal's objective should include hiding their crime while achieving a personal goal
- Non-criminal players should have objectives that may conflict with finding the truth
- public_info is NOT a repeat of the character profile. It is situation-specific information that all players can see:
  - Why this character came to this gathering / event
  - Who they arrived with or who invited them
  - Their publicly known relationship to the victim or other characters
  - What they were seen doing before the incident
  - Their stated reason for being here
  - Example: "被害者の古い友人として招待された。田中健二と一緒に来館し、到着時にロビーで他の客と談笑していた。夕食前に被害者と二人で書斎に入るのが目撃されている。"
  - Each public_info should mention at least one other character by name

## Constraints

- Exactly one player must be the criminal
- The criminal must be one of the player characters
- Each player must have a unique secret and objective
- The mystery must be solvable through the clues provided
- Include 3 phases: investigation, discussion, voting
- Investigation phase should reference 4-6 locations from the map
- All text content must be in Japanese
- The map layout must support the crime's logic (alibis, escape routes, evidence placement)
