## Map Layout Rules

Think of the map as a grid (graph paper). Each location occupies a rectangle on this grid.

- **x, y**: top-left corner of the location on the grid (integers, 0-based)
- **w, h**: width and height in grid cells (minimum 1)
- **area_type**: "indoor", "outdoor", or "semi_outdoor"
- **Locations MUST NOT overlap** on the grid
- Place locations so that connected rooms are adjacent on the grid
- Use y=0 row for outdoor areas (garden, parking, etc.)
- Use y=1+ rows for indoor areas (1F, 2F, etc.)
- Total grid should fit within roughly 6x5 cells

Connection side indicates where the passage is on the "from" room:
- "north", "south", "east", "west" (which wall the door/window is on)

## Map Guidelines

- Generate 6-10 locations that form a coherent floor plan
- Each location must have 2-4 features (objects, furniture, notable details)
- Connections define how locations are linked: "door", "corridor", "stairs", "window", "hidden_passage"
- The map must be spatially consistent (no impossible layouts)
- Include at least one location that is lockable or restricted
- The crime scene should be connected to at least 2 other locations
- Consider sight lines and sound: adjacent rooms can hear loud noises
- investigation_locations in phases reference location IDs from the map

## Constraints

- Exactly one player must be the criminal
- The criminal must be one of the player characters
- Each player must have a unique secret and objective
- The mystery must be solvable through the clues provided
- Include 3 phases: investigation, discussion, voting
- Investigation phase should reference 4-6 locations from the map
- All text content must be in Japanese
- The map layout must support the crime's logic (alibis, escape routes, evidence placement)
