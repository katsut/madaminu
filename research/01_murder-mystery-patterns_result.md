# マーダーミステリーにおけるシナリオ構造設計と自動生成システムの要件定義：4〜7人プレイに向けた50のパターン・テンプレート構築論

マーダーミステリー（以下、マダミス）という遊戯形態は、プレイヤーが物語の登場人物となり、殺人事件の真相を解明する、あるいは犯人として逃げ切ることを目的とした体験型推理ゲームである。この遊戯は「一生に一度しか遊べない」という不可逆性を持ち、デジタルプラットフォームへの移行において、この制約をいかに技術的・構造的に解消するかが最大の焦点となる。特に、4〜7人のプレイヤーがリアルタイムで参加するオンラインアプリの開発において、シナリオの自動生成を実現するためには、物語を単なる「固定されたテキスト」としてではなく、組み替え可能な「動的なシステム」として再定義する必要がある。本報告書では、シナリオの基本構造から、舞台設定、動機、トリック、キャラクター役割、エンディング設計、そしてリプレイ性の確保に至るまでを包括的に調査し、自動生成の基盤となる50のシナリオテンプレートを構築するための知識体系を提示する。

## 1. シナリオの基本構造とフェーズ進行の設計

シナリオの自動生成を試みる際、まず理解すべきはマダミスの構造が「情報の非対称性」と「時間軸の管理」によって成立している点である。物語の「起承転結」をゲームデザインとしていかに落とし込むかが、プレイヤーの没入感を左右する 。  

### 1.1 シナリオを構成する基本要素

マダミスのシナリオは、静的なテキストデータと動的なゲーム内アクションの組み合わせで構成される。主要な構成要素は以下の通りである。

| 構成要素 | 内容と役割 | 生成システムにおける変数 |
| --- | --- | --- |
| **導入（イントロダクション）** | 世界観、事件の概要、プレイヤーの初期状況を提示する 。 | 舞台設定、被害者の属性、集合目的。 |
| **キャラクターシート（ハンドアウト）** | 各プレイヤーの役割、秘密、目的、行動履歴（タイムライン）を記述する 。 | 秘密の重要度、他者との関係性、嘘の許容範囲。 |
| **調査・証拠カード** | 現場や所持品から得られる情報の断片。トリック解明の鍵となる 。 | 発見難易度、情報の真偽、所有権。 |
| **フェーズ進行ルール** | 議論時間や情報の開示順序を制御するシステム 。 | タイマー、イベントフラグ、投票タイミング。 |
| **エンディング（解決編）** | 真相の提示と各キャラクターの後日談 。 | 投票結果による分岐、ミッション達成度。 |

自動生成システムにおいて、これらの要素は独立したモジュールとして設計されるべきである。特に「結末（真相）」から逆算して、各要素に矛盾がないよう証拠やタイムラインを配置する手法が、物語の堅牢性を確保する上で最も効果的である 。  

### 1.2 フェーズ進行のバリエーションと設計手法

オンラインアプリにおいては、物理的な制約がないため、多様な進行制御が可能となる。

-   **タイマー制（固定時間進行）：** 最も一般的な手法。プレイヤー間に緊張感を与え、議論の優先順位を強制的に選択させる。自動生成においては、参加人数や情報量に応じて最適な議論時間を算出するアルゴリズムが必要となる。

-   **イベント駆動型（トリガー進行）：** 特定の証拠が発見された、あるいは特定のプレイヤーが「告白」を行った際に、物語が次のステップへ進む手法 。これは「転」を意図的に作り出すのに適しており、物語の密度を高める効果がある。  

-   **フリー進行（プレイヤー主導）：** プレイヤーの合意によってフェーズを移行させる。自由度は高いが、議論の停滞を招くリスクがあるため、アプリ側でのナビゲーションが重要となる。

-   **非線形型（プロシージャル・ナラティブ）：** プレイヤーの選択によって、次に発生するイベントや開示される情報が動的に変化する手法 。  

### 1.3 犯人決定ロジックの二面性

「犯人が最初から決まっているパターン」と「プレイ中の行動で犯人が変わるパターン」の設計手法には、根本的な違いが存在する。

1.  **固定型設計：** 全ての伏線が特定の人物（犯人）に収束するように設計される。高い物語性と、緻密なロジックパズルを提供できる。自動生成では、犯人Aに対して「アリバイがない時間帯」と「殺害可能な凶器」を割り当て、他者には「ミスリードとなる不審な行動」を割り振る 。  

2.  **動的可変型（事後確定型）設計：** プレイ開始時点では「殺意」や「機会」を全員が持っており、最終的な「決定的な一撃」を誰が放ったかを、調査フェーズでの行動やカードの引きによって確定させる 。これを実現するためには、全キャラクターに「犯人になり得るタイムライン」と「被害者を殺害するのに十分な動機」を等しく付与する必要がある 。  

## 2. 舞台設定の類型と空間的制約

舞台設定は、プレイヤーの行動範囲を限定し、情報の流れを制御するための「空間的フィルター」として機能する 。  

### 2.1 舞台設定の類型と特徴

マダミスで頻用される舞台は、外部からの隔離性（クローズド・サークル）の度合いによって分類できる。

| 舞台設定 | 構造的特徴 | ゲームデザインへの影響 |
| --- | --- | --- |
| **洋館・古宅** | 部屋が仕切られ、見取り図が重要。 | 移動履歴によるアリバイ工作が主眼となる 。 |
| **孤島・吹雪の山荘** | 外部犯の可能性を100%排除。 | プレイヤー間の相互不信を最大化させる。 |
| **列車・宇宙船** | 場所の移動が不可逆的で、時間制限が明確。 | タイムリミットによる焦燥感と、移動制約の活用。 |
| **学校・病院** | 職業や学年などの「階層構造」が存在する。 | 権限による情報格差や、主従関係の強調 。 |
| **ファンタジー世界** | 魔法や特殊能力が存在する 。 | 物理法則を無視したトリックの導入が可能 。 |

### 2.2 舞台設定が物語構造に与える影響

舞台設定は単なる背景ではなく、以下の3つの制約を通じてゲーム性を規定する。

-   **密室性：** 物理的な密室だけでなく、心理的・社会的密室（例：秘密結社の集会）も含まれる。これにより「犯人はこの中にしかいない」という論理的前提が成立する。

-   **移動制約：** 誰がどの部屋にいたか、という情報を管理することで、アリバイの構築と崩壊を制御する。オンラインアプリでは、見取り図を常に画面の一部に配置し、視覚的に把握させることが重要である 。  

-   **コミュニケーション制約：** 「密談」の可否や、特定ペアでしか共有できない情報の存在が、情報の非対称性を生む。デジタルならではの演出として、特定の場所（部屋）にいるプレイヤー同士しか音声会話ができないといった仕組みが考えられる 。  

## 3. 動機の類型とコンフリクトの多層化設計

動機はキャラクターの行動原理（Why）であり、プレイヤーがその役を「演じる」ための強力なインセンティブとなる 。  

### 3.1 殺人動機のパターン分類

自動生成システムにおいて、動機はキャラクターの属性と過去の出来事を結びつける「関係性のタグ」として扱われる。

### 3.2 「全員に動機がある」状態の構築

推理ゲームとしての難易度とドラマ性を両立させるには、被害者を中心とした「負の人間関係」を網状に配置する必要がある。

-   **サブプロットの挿入：** 殺人事件とは別に、盗難や不倫、ビジネスの不正などの「小さな秘密」を全員に持たせる 。これにより、犯人ではないプレイヤーも「自分の秘密を隠すために嘘をつく」必要性が生じ、議論が複雑化する 。  

-   **ミッシングリンクの活用：** 一見無関係なキャラクター同士が、裏で共通の過去や秘密（例：10年前の事故の共犯者）で繋がっている構造。これにより、中盤以降に人間関係が劇的に再定義される「転」を演出できる 。  

-   **利害関係の逆転：** 被害者が死ぬことで得をする人物（遺産相続人）と、被害者が生きていることで得をしていた人物（被保護者）を混在させ、疑いの矛先を分散させる。

## 4. トリック・ギミックの類型とデジタル特有の手法

トリックはミステリーの華であり、自動生成においては「物理法則」と「情報の提示順序」をいかに操作するかが重要となる。

### 4.1 トリックの分類一覧

| トリック種別 | 内容  | 実装上の注意点 |
| --- | --- | --- |
| **物理トリック** | 密室、凶器の隠蔽、死体移動。 | 図解やアイテムの移動履歴で矛盾を証明できるようにする 。 |
| **時間トリック** | アリバイ偽装、死亡推定時刻の操作。 | タイムラインの整合性をシステム側で厳密にチェックする 。 |
| **情報トリック** | 嘘の証言、ダイイングメッセージの改ざん。 | 証拠品が「真実」を語る役割を持たせる 。 |
| **メタトリック** | ルール自体の悪用、GMの正体。 | 一度しか使えない驚きを生むが、連発は禁物。 |

### 4.2 デジタルゲーム特有のトリックと演出

オンラインアプリ開発において、アナログでは不可能なギミックを導入することが、競合サービスとの差別化に繋がる 。  

-   **UI/UXの動的操作：** プレイヤーの画面に表示される情報（体力ゲージ、所持品、通知メッセージ）が、特定の条件下で密かに書き換えられる。あるいは、特定のキャラクターだけに見える「偽のシステム通知」を送る。

-   **音声・映像のディープフェイク：** アプリ内で共有されるボイスメッセージやビデオログが、AIによって改ざんされている可能性をルールに組み込む。

-   **ログ・ハッキング：** チャット履歴や調査ログを、特定のスキルを持つキャラクターが編集・削除できる仕組み 。  

-   **リアルタイム環境変化：** 議論中に「停電」が発生し、特定のプレイヤーの画面が暗転している間に、アイテムの受け渡しや死体移動がシステム上で自動実行される。

## 5. キャラクター役割の類型と人数別配分

キャラクターはプレイヤーが物語を体験するための「インターフェース」であり、各役職には明確なゲーム上の機能（情報の保持・開示）が割り振られる 。  

### 5.1 典型的な役職とキャラクター設計

| 役割名 | ゲーム上の機能 | 秘密の設計パターン |
| --- | --- | --- |
| **犯人（クロ）** | 議論を混乱させ、逃げ切る。 | 犯行手段、本当の動機、証拠の隠し場所 。 |
| **探偵（シロ）** | 議論を整理し、真相に導く。 | 自身の過去の過ち、被害者への個人的な恩義 。 |
| **共犯者** | 犯人を助け、疑いを逸らす。 | 犯人との秘密の繋がり、協力の見返り 。 |
| **目撃者** | 部分的な真実を知っている。 | 見たことを隠さなければならない理由（自身の不法行為など）。 |
| **不審者・容疑者** | 誤認逮捕を誘うミスリード役。 | 殺人とは無関係な重い罪、奇妙な性癖 。 |
| **被害者の関係者** | 感情的な揺さぶりをかける。 | 被害者への愛憎、隠された血縁関係 。 |

### 5.2 4人〜7人の人数別最適な役割配分

人数が増えるほど、情報の流通経路が複雑になり、サブストーリーの重要性が増す。

| 人数  | 推奨構成テンプレート | 構造的特徴 |
| --- | --- | --- |
| **4人** | 犯人1、探偵役1、重要目撃者1、不審者1。 | 全員の密着度が高く、一人でも嘘がバレると崩壊しやすい。濃密な議論向け 。 |
| **5人** | 犯人1、共犯者1、探偵役1、関係者2。 | 共犯者の存在により、多数決のコントロールが可能になる。心理戦の比重が増す 。 |
| **6人** | 犯人1、共犯者1、第三勢力1、関係者3。 | 複数の小グループ（派閥）が発生し、情報共有の分断が起きやすくなる 。 |
| **7人** | 犯人1、共犯者1、探偵役1、複雑な関係者4。 | 複数の動機が絡み合い、情報の取捨選択が極めて難しくなる 。 |

### 5.3 キャラクター間の関係性パターン

キャラクター同士を繋ぐ「線」の設計は、自動生成における関係性グラフの構築に不可欠である。

1.  **対立軸：** 敵対企業、ライバル、遺産争い。

2.  **愛情・執着軸：** 恋人、片思い、ストーカー 。  

3.  **主従・依存軸：** 社長と秘書、アイドルとファン、師匠と弟子 。  

4.  **血縁軸：** 親子、絶縁した兄弟、隠し子。

5.  **過去の因縁軸：** 同じ事件の生存者、過去の共犯者、初恋の相手。

これらの関係性は、表面上の「公称関係」と、ハンドアウトでのみ明かされる「真実の関係」の二重構造にすることで、情報の開示によるカタルシスを生む 。  

## 6. エンディング分岐と満足度の最大化

エンディングはプレイヤーにとっての報酬であり、その設計如何でゲームの評価が決定する 。  

### 6.1 マルチエンディングの分岐条件

単なる「犯人の特定」以外に、以下の要素を分岐条件に組み込むことで、物語に深みを持たせることができる。

-   **投票結果：** 最多票を集めた人物が誰か。

-   **ミッション達成度：** 「秘密を隠し通したか」「特定の証拠を破壊したか」「意中の人物を守ったか」 。  

-   **最終選択：** 全ての議論が終わった後に提示される究極の二択（例：犯人を警察に突き出すか、見逃すか） 。  

-   **信頼関係の数値化：** ゲーム中のプレイヤー間の協力行動やギフト送信を「信頼度」として蓄積し、エンディングに反映させる。

### 6.2 全員が満足できるエンディング設計の工夫

敗北（犯人として捕まる、あるいは犯人を逃す）したプレイヤーにも、「納得感」と「物語的充足感」を与える必要がある。

-   **個別後日談の用意：** 全体の勝敗とは別に、各キャラクターがその後どのような人生を歩んだかを詳細に描く。ミッションの成否によって、この文章を変化させる 。  

-   **解釈違いの防止：** エンディングでキャラクターがプレイヤーの意図しない行動（例：急に改心する）を取らないよう、性格設定とエンディングの整合性を保つ 。  

-   **真相のグラデーション：** 「100%の正解」以外にも、一部の謎が残るエンディングや、隠された真の黒幕を示唆するトゥルーエンドを用意する。

## 7. リプレイ性の確保と自動生成の変数設計

同じ「型」を使いながら、異なる体験を提供するための技術的アプローチについて考察する。

### 7.1 リプレイ性を生むための変数設計

1.  **犯人のシャッフル：** 5W1Hの「誰が」を毎回変更する。これには、全キャラクターに殺害機会と動機を割り振るロジックが必要である 。  

2.  **凶器とトリックのペアリング：** 毒殺、刺殺、絞殺などのバリエーションと、それに対応する証拠品を動的に配置する。

3.  **情報の分散アルゴリズム：** どの情報を誰に持たせるかをランダム化する。ただし、重要な証拠が一箇所に固まらないよう分散させる必要がある。

4.  **サブストーリーの挿入：** 30種類程度のサブシナリオから、各プレイヤーに2〜3個をランダムに割り振る。

### 7.2 自動生成における一貫性の担保

プロシージャル・ナラティブにおいて最も困難なのは、生成された物語の「論理的整合性」である。

-   **前提条件の木構造：** 「証拠Aが見つかったならば、発言Bが可能になる」といった条件分岐を厳密に定義する 。  

-   **プランニング・エンジン：** AIが初期状態からゴール（解決）までの道筋をシミュレートし、詰みが発生しないことを確認した上でシナリオを出力する 。  

-   **文脈の維持：** 大規模言語モデル（LLM）を活用し、舞台設定に合わせた文体や台詞回しを動的に生成するが、キャラクターの核心となる設定（秘密）は不変のデータとして保持する。

## 8. シナリオ自動生成のための50パターン・テンプレートカタログ

以下に、4〜7人プレイに対応し、自動生成のベースとして活用可能な50のシナリオテンプレートを提示する。これらは「ジャンル」「舞台」「核心トリック」「主要コンフリクト」の組み合わせによって定義される。

### A. クラシック・ミステリー（10パターン）

1.  **吹雪の山荘と遺言状：** 外部遮断×遺産相続×アリバイ工作。

2.  **孤島の芸術家村：** クローズドサークル×見立て殺人×芸術的動機。

3.  **豪華列車の密室：** 移動制限×時刻表トリック×復讐劇。

4.  **嵐の夜の古宅：** 停電ギミック×凶器の消失×秘密の地下室。

5.  **霧のロンドン・パブ：** 偶然の集まり×過去の強盗事件×正体隠蔽。

6.  **仮面舞踏会の悲劇：** 全員仮面（素性不明）×身代わりトリック×愛憎。

7.  **図書館の秘密通路：** 物理パズル×隠された古文書×知識の競合。

8.  **廃村の古い儀式：** 土着信仰×毒物混入×村人の共謀。

9.  **名探偵の葬儀：** 被害者が探偵×残された弟子の推理戦×遺産争い。

10.  **劇場での死：** 演劇中の事故装い×照明トリック×舞台裏の人間関係。

### B. SF・近未来（10パターン）

11.  **宇宙船の酸素漏れ：** 資源制限×生命維持装置の破壊×裏切り。

12.  **電脳世界のバグ：** アバター殺人×ログ改ざん×AIの暴走。

13.  **タイムリープのループ：** 時間遡行×記憶の保持×運命の回避。

14.  **クローン工場の死体：** 自己同一性×個体識別トリック×科学倫理。

15.  **火星基地の沈黙：** 閉鎖環境×通信ラグ×異星物質の感染。
